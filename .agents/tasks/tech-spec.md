# Tech Spec: Pre-Commit Hook Integration (US-007)

## Overview

This spec covers the `ks install-hook` command, which writes a git pre-commit hook to
`.git/hooks/pre-commit`. The hook runs `ks snapshot && ks diff` on every commit, emits
structured diff JSON to stdout (advisory only, always exits 0), auto-starts Chrome when
necessary, and degrades gracefully if project URLs are unreachable.

**Story**: US-007
**PRD**: Snapshot History and Regression Detection
**Acceptance criteria**: See PRD story block.

---

## Dependencies on Other Stories

US-007 depends on commands introduced in earlier stories in the same PRD:

| Command | Source story | Role in hook |
|---|---|---|
| `ks snapshot` | US-001/US-002 (snapshot capture) | Takes a new snapshot before the commit |
| `ks diff` | US-005/US-006 (diff engine) | Compares latest snapshot against baseline |
| `ks start` | existing (`cmd/start.go`) | Auto-starts Chrome if not running |

The `.ks-project.json` file (project config introduced in US-001) must exist in the repo
root. `install-hook` validates this at install time.

## Architecture Overview

```
ks install-hook
     │
     ├── validates .ks-project.json exists in CWD
     ├── checks .git/hooks/pre-commit existence → warns if present, aborts
     └── writes executable shell script to .git/hooks/pre-commit

.git/hooks/pre-commit (shell script)
     │
     ├── ks start --local (no-op if Chrome already running)
     ├── ks snapshot       → JSON to stdout
     ├── ks diff           → JSON to stdout (advisory)
     └── exit 0 always
```

The hook itself is a **pure shell script** embedded in the Go binary. The Go code only
writes and validates it; all regression logic lives in `ks snapshot` and `ks diff`.

## Component Design

### 1. `cmd/install_hook.go`

**File**: `/workspace/cmd/install_hook.go`
**Package**: `cmd`
**Exported function**: `RunInstallHook(args []string)`

#### Responsibilities

1. Validate that `.ks-project.json` exists in the current working directory.
2. Validate that `.git/` exists (i.e. the CWD is a git repository root or nested in one).
3. Resolve the git root (walk up from CWD until `.git/` is found, max 10 levels).
4. Check if `.git/hooks/pre-commit` already exists.
   - If it does **and** `--force` is NOT passed: emit `output.Fail` with a warning and exit 2.
   - If it does **and** `--force` IS passed: overwrite silently.
5. Ensure `.git/hooks/` directory exists (create if missing).
6. Write the embedded hook script to `.git/hooks/pre-commit` with mode `0755`.
7. Emit `output.Success` with the path written.

#### Function signature

```go
func RunInstallHook(args []string) {
    force := hasFlag(args, "--force")
    // ... implementation ...
}
```

#### Output on success

```json
{
  "ok": true,
  "command": "install-hook",
  "result": {
    "hook": ".git/hooks/pre-commit",
    "message": "pre-commit hook installed. Run 'ks diff' after each commit to review regressions."
  }
}
```

#### Output on error (hook already exists, no --force)

```json
{
  "ok": false,
  "command": "install-hook",
  "error": "pre-commit hook already exists at .git/hooks/pre-commit",
  "hint": "Use --force to overwrite the existing hook."
}
```

#### Output on error (no .ks-project.json)

```json
{
  "ok": false,
  "command": "install-hook",
  "error": "no .ks-project.json found in current directory",
  "hint": "Run 'ks init' to create a project config before installing the hook."
}
```

### 2. Hook Script Content

The hook is a constant string embedded directly in `install_hook.go` (no `//go:embed`
needed — it is a short, static string).

```bash
#!/bin/sh
# kaleidoscope pre-commit hook — auto-generated by 'ks install-hook'
# Advisory only: always exits 0. Output is structured JSON for agent consumption.

set -e

# Auto-start Chrome if not running (no-op when already up)
ks start --local 2>/dev/null || true

# Capture snapshot of all project URLs
SNAPSHOT_OUT=$(ks snapshot 2>&1)
echo "$SNAPSHOT_OUT"

# Run diff against baseline; output JSON regardless of result
DIFF_OUT=$(ks diff 2>&1)
echo "$DIFF_OUT"

# Always exit 0 — advisory only, the agent decides
exit 0
```

**Design notes**:
- `set -e` applies to the script body but individual commands use `|| true` where
  graceful failure is required (e.g. `ks start`).
- If `ks snapshot` or `ks diff` fail (unreachable URLs, Chrome down), their `output.Fail`
  JSON is still forwarded to stdout so the agent can inspect the error.
- The script never calls `exit 1` or any non-zero exit.

## API Definitions

`ks install-hook` is a CLI command, not an HTTP endpoint. Its interface is:

| Aspect | Value |
|---|---|
| CLI invocation | `ks install-hook [--force]` |
| Stdout | `output.Result` JSON (one line) |
| Stderr | (none — errors go through `output.Fail` to stdout) |
| Exit code success | `0` |
| Exit code failure | `2` (consistent with all other `ks` commands) |

---

## Security Considerations

1. **File permissions**: The hook script must be written with mode `0755`. Using `0644`
   would create a hook that git silently ignores (not executable).

2. **Script injection**: The hook content is a compile-time constant. No user-supplied
   data is interpolated into the script body, so there is no injection risk.

3. **Overwrite protection**: Without `--force`, the command refuses to overwrite an
   existing hook. This prevents silently clobbering a developer's custom pre-commit hook
   (e.g. linters, formatters, secret scanners).

4. **Always-exit-0 contract**: The hook must never exit non-zero. A non-zero exit from
   a pre-commit hook blocks the commit entirely. Since kaleidoscope is advisory, blocking
   commits is out of scope and would break developer workflow.

5. **Git root resolution**: The command walks up to find `.git/` rather than assuming
   CWD is the repo root, preventing writing to an incorrect location when invoked from a
   subdirectory. Maximum walk depth is capped at 10 levels to avoid infinite loops on
   misconfigured filesystems.

## Error Cases & Graceful Degradation

| Condition | Behavior |
|---|---|
| `.ks-project.json` missing | `output.Fail` + exit 2 |
| Not inside a git repo | `output.Fail` + exit 2 |
| Hook already exists (no `--force`) | `output.Fail` + exit 2 (with hint) |
| Hook already exists (`--force`) | Overwrite silently, `output.Success` |
| Cannot write hook (permissions) | `output.Fail` wrapping `os` error + exit 2 |
| Chrome not running (at hook runtime) | `ks start --local` is called; `|| true` ensures non-fatal |
| URLs unreachable (at hook runtime) | `ks snapshot` emits `output.Fail` JSON to stdout; hook exits 0 |
| `ks diff` finds regressions | `ks diff` emits JSON with regression details; hook exits 0 |

## Testing Approach

Per project quality gates: `go test ./...`

Recommended test cases for `cmd/install_hook_test.go`:

1. **Happy path**: creates `.ks-project.json` and a temp git repo, runs `RunInstallHook`,
   asserts file exists at `.git/hooks/pre-commit`, has mode `0755`, and contains expected
   shebang + `ks snapshot` + `ks diff`.

2. **No project config**: no `.ks-project.json` → output JSON has `ok: false`, exit 2.

3. **Not a git repo**: no `.git/` directory → output JSON has `ok: false`, exit 2.

4. **Hook exists, no force**: pre-existing hook → `ok: false`, hint mentions `--force`.

5. **Hook exists, with force**: `--force` flag → hook overwritten, `ok: true`.

6. **Hook content is executable**: after install, `os.Stat` shows `0755` permissions.
