# Implementation Plan: Pre-Commit Hook Integration (US-007)

## Overview

Implements `ks install-hook`, a CLI command that writes an executable git pre-commit hook
to `.git/hooks/pre-commit`. The hook runs `ks snapshot && ks diff` advisory-only (always
exits 0) and auto-starts Chrome if needed.

**Files to create**: `cmd/install_hook.go`, `cmd/install_hook_test.go`
**Files to modify**: `cmd/usage.go`, `main.go`
**Quality gate**: `go test ./...`

## Phase 1 — Core Command Implementation

### Task 1.1 — Create `cmd/install_hook.go`

**File**: `/workspace/cmd/install_hook.go`

#### Sub-task 1.1.1 — Package declaration and imports
- Declare `package cmd`
- Import: `errors`, `fmt`, `os`, `path/filepath` (for git-root walk), plus local
  `kaleidoscope/output`

#### Sub-task 1.1.2 — Define the hook script constant
- Define `hookScript` as a `const string` containing the exact shell script from the
  tech spec:
  ```sh
  #!/bin/sh
  # kaleidoscope pre-commit hook — auto-generated by 'ks install-hook'
  # Advisory only: always exits 0. Output is structured JSON for agent consumption.

  set -e

  # Auto-start Chrome if not running (no-op when already up)
  ks start --local 2>/dev/null || true

  # Capture snapshot of all project URLs
  SNAPSHOT_OUT=$(ks snapshot 2>&1)
  echo "$SNAPSHOT_OUT"

  # Run diff against baseline; output JSON regardless of result
  DIFF_OUT=$(ks diff 2>&1)
  echo "$DIFF_OUT"

  # Always exit 0 — advisory only, the agent decides
  exit 0
  ```
- No user data is interpolated; the constant is entirely static (no injection risk)

#### Sub-task 1.1.3 — Implement `findGitRoot(startDir string) (string, error)`
- Walk upward from `startDir`, checking for `.git/` directory at each level
- Cap walk at 10 levels to avoid infinite loops on misconfigured filesystems
- Return the directory that contains `.git/`, or an error if not found within limit

#### Sub-task 1.1.4 — Implement `hasFlag` usage (reuse existing `cmd/util.go` helper)
- Confirm `hasFlag(args []string, flag string) bool` already exists in `cmd/util.go`
- If present, use it directly; do not duplicate

#### Sub-task 1.1.5 — Implement `RunInstallHook(args []string)`
Ordered steps inside the function:

1. **Parse flags**: `force := hasFlag(args, "--force")`
2. **Validate `.ks-project.json`**:
   - `os.Stat(".ks-project.json")` in CWD
   - On error: call `output.Fail("install-hook", err, "Run 'ks init' to create a project config before installing the hook.")` then `os.Exit(2)`
3. **Find git root**:
   - Call `findGitRoot(cwd)`
   - On error: call `output.Fail` with message "not inside a git repository" then `os.Exit(2)`
4. **Resolve hook path**: `hookPath = filepath.Join(gitRoot, ".git", "hooks", "pre-commit")`
5. **Check for existing hook** (when `!force`):
   - `os.Stat(hookPath)` — if it exists and `!force`:
     `output.Fail("install-hook", errors.New("pre-commit hook already exists at "+hookPath), "Use --force to overwrite the existing hook.")` then `os.Exit(2)`
6. **Ensure hooks directory exists**:
   - `os.MkdirAll(filepath.Dir(hookPath), 0755)`
7. **Write hook script**:
   - `os.WriteFile(hookPath, []byte(hookScript), 0755)`
   - On error: `output.Fail` wrapping the OS error, then `os.Exit(2)`
8. **Emit success**:
   ```go
   output.Success("install-hook", map[string]any{
       "hook":    hookPath,
       "message": "pre-commit hook installed. Run 'ks diff' after each commit to review regressions.",
   })
   ```

### Task 1.3 — Add usage documentation in `cmd/usage.go`

**File**: `/workspace/cmd/usage.go`

#### Sub-task 1.3.1 — Add `"install-hook"` key to `CommandUsage` map
Insert the full usage entry as specified in the tech spec:
```go
"install-hook": `ks install-hook — Install a git pre-commit hook for automatic regression checks

Usage: ks install-hook [--force]

Options:
  --force    Overwrite an existing pre-commit hook without warning

Description:
  Writes an executable shell script to .git/hooks/pre-commit that runs
  'ks snapshot' and 'ks diff' before every commit. Output is structured
  JSON sent to stdout for agent consumption. The hook always exits 0
  (advisory/non-blocking) so it never blocks a commit.

  Requires .ks-project.json to exist in the current directory.
  Chrome is auto-started if not already running.
  If project URLs are unreachable the hook fails gracefully and logs a
  structured error to stdout.

Examples:
  ks install-hook             # Install hook (fails if one already exists)
  ks install-hook --force     # Install hook, overwriting any existing hook

Notes:
  - .ks-project.json is committed to the repo; .kaleidoscope/ is gitignored.
  - The hook is advisory: it never blocks a commit regardless of diff results.
  - The agent reads stdout JSON to decide whether to alert the developer.
`,
```

## Dependency Notes

- US-007 depends on `ks snapshot` (US-001/US-002) and `ks diff` (US-005/US-006) at
  **runtime** (inside the hook script), but the install-hook command itself has no
  compile-time dependency on those packages — it only writes a shell script string.
- The `hasFlag` helper in `cmd/util.go` must be confirmed present before Task 1.1.4.
- The `output.Fail` / `output.Success` signatures follow the pattern established in
  `output/format.go` and used throughout the existing `cmd/` files.
